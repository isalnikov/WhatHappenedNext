# План разработки Телеграм бота "WhatHappenedNext"

## Описание проекта
Интеллектуальный ИИ веб-агент на Java 25 + Spring Boot 4, который:
- Получает и хранит список сайтов и каналов для парсинга
- Парсит новости с указанных источников
- Сохраняет новости в базу данных H2
- Анализирует новости через qwen code cli (данные передаются через stdin)
- Отправляет результат анализа пользователю в Telegram

## Принципы разработки

### Методология
- **TDD (Test-Driven Development)**: Сначала тест, потом код
- **ContractFirst**: Разработка начинается с определения контрактов (API, интерфейсы)
- **Context Engineering**: План структурирован для эффективной работы AI-агентов

### Паттерны и принципы
- KISS (Keep It Simple, Stupid)
- DRY (Don't Repeat Yourself)
- YAGNI (You Aren't Gonna Need It)
- BDUF (Big Design Up Front)
- SOLID (Single Responsibility, Open-Closed, Liskov Substitution, Interface Segregation, Dependency Inversion)
- APO (Architecture Pattern Optimization)
- Бритва Оккама

### Технические требования
- **Java 25**: Использование всех современных фич:
  - Виртуальные потоки (Project Loom)
  - Pattern Matching
  - Records
  - Sealed-классы
- **Логирование**: Во всех классах
- **Покрытие тестами**: Не менее 90%
- **Без Lombok**: Чистый код без магических аннотаций

### Архитектурные решения
- **База данных**: H2 (in-memory для тестов, file-based для продакшена)
- **AI интеграция**: qwen code cli через stdin (безопасная передача больших данных)
- **Асинхронность**: Виртуальные потоки для обработки большого количества данных

## Примечания

- Все классы должны иметь логирование через SLF4J
- Использовать Records для DTO и доменных моделей
- Использовать sealed-классы для результатов операций
- Избегать Lombok - писать геттеры/сеттеры явно
- Применять виртуальные потоки для I/O операций
- Данные для AI передавать только через stdin
